# ChatGPT Stabilizer Pro

<p align="center">
  <a href="./README.en.md"><strong>English</strong></a>
</p>

ChatGPT Stabilizer Pro 是一款面向 ChatGPT 长对话场景的稳定、高效、低干预桌面端浏览器扩展。它通过多档位优化策略缓解长对话带来的滚动卡顿、输入延迟和页面无响应问题，并尽量保持 ChatGPT 原生交互体验。

## 功能特性

- 默认识别 `chatgpt.com` 和 `chat.openai.com` 两个域名
- 提供 `Off`、`Monitor`、`Standard`、`Performance` 四个可选模式，以及一个已注册但暂不可选的 `Extreme` 计划模式
- 内置页面控制面板，可查看运行状态、优化统计、诊断信息并手动重新同步。
- 支持桌面 Chrome/Chromium 系 Manifest V3 扩展加载。
- 支持桌面 Firefox 临时扩展加载。
- 目前支持英文和简体中文语言，插件会根据当前浏览器的语言自动加载对应语言包

## 本地安装

先从 GitHub Releases 下载最新的 `ChatGPT-Stabilizer-Pro-v*.zip`，然后把 zip 解压到一个固定目录。浏览器本地加载时选择的是解压后的文件夹，不是 zip 文件本身。

### Chrome 或 Edge 浏览器

1. 打开 `chrome://extensions/` 或 `edge://extensions/`
2. 开启右上角的 `开发者模式`。
3. 点击 `加载已解压的扩展程序`。
4. 选择刚才解压出来的扩展文件夹，文件夹第一层应包含 `manifest.json`。
5. 打开或刷新 ChatGPT 页面。

### Firefox

1. 打开 `about:debugging#/runtime/this-firefox`。
2. 点击 `临时载入附加组件`。
3. 选择刚才解压出来的扩展文件夹中的 `manifest.json`。
4. 打开或刷新 ChatGPT 页面。

## 移动端

当前版本不正式支持移动端浏览器，主要面向桌面浏览器使用。

## 控制面板按钮

- 复制：复制当前面板信息，方便反馈问题。
- 刷新：手动重新同步当前页面状态。
- 恢复当前会话：如果遇到预期之外的问题，可以尝试恢复当前会话，清理当前会话的优化状态，并回到更安全的模式。
- 收起面板：只保留简略的信息显示。
- 隐藏面板：隐藏面板和简略信息板，保留最小的徽章。

## 控制面板选项卡

控制面板包含几个常用选项卡：

- 总览：查看当前运行状态、目标模式、实际模式、优化率和关键统计。
- 优化等级：切换 `Off`、`Monitor`、`Standard`、`Performance` 等优化模式，并查看每个模式的说明和风险提示。
- 聊天统计：查看消息总数、内容块数量、已观测内容、可优化内容、已优化内容和当前可见范围。
- 性能指标：查看初始化、同步、重同步耗时，以及 `Performance` 模式下的折叠、恢复和阻塞统计。
- 最近事件：查看插件近期运行事件，并在排查问题时开始调试、停止调试、复制或导出脱敏后的调试记录。

## 优化模式

插件提供多个运行模式。普通使用建议从 `Standard` 开始。

### Off

关闭全部插件优化、统计、诊断、事件和调试记录，仅保留控制面板本身可操作。

适合：

- 临时排查问题
- 想完全恢复 ChatGPT 原生页面行为
- 怀疑插件影响页面交互时做对照测试

### Monitor

关闭页面优化，但保留页面识别、消息统计、诊断、最近事件和调试记录。

适合：

- 查看页面识别是否正常
- 查看消息统计和诊断信息
- 在不改变页面内容行为的情况下收集状态
- 在无优化接管的前提下录制调试记录

### Standard

默认推荐模式。

它会对离开当前视野的内容做保守优化，不影响复制、选择、点击链接、编辑、搜索等原生操作。

适合：

- 大多数长对话
- 希望稳定优先，尽可能避免影响原生页面
- 页面有轻微到中等卡顿

### Performance

性能优先模式。

它会对远处的高收益历史内容做原位折叠，等内容接近当前视野时再恢复。这个模式收益更高，但历史内容第一次复制、选择或点击前，可能需要先等待短暂恢复。

适合：

- 超长对话
- 大量代码块、表格、Markdown、图片或长文本导致页面明显卡顿
- `Standard` 模式收益不够时

注意：

- 折叠的历史内容仍在原位置，不会被删除。
- 接近视野、点击、选择、聚焦时会自动恢复。

### Extreme

一个比 `Performance` 更激进的优化档位，计划在下个版本推出，当前仍暂不可选。

## 推荐用法

日常使用：

1. 先使用 `Standard`。
2. 如果长对话仍明显卡顿，再切换到 `Performance`。
3. 如果怀疑插件导致问题，切到 `Off` 做对照。
4. 如果只想查看状态或录制无优化调试记录，切到 `Monitor`。

长对话明显卡顿时：

1. 打开控制面板。
2. 切换到 `Performance`。
3. 等待面板中的优化状态稳定。


## 权限与隐私

完整隐私政策请查看：[PRIVACY.md](./PRIVACY.md#中文版本)。

本插件仅申请本地 `storage` （存储）权限，无任何联网功能，并且内容脚本只在以下页面运行：

- `https://chat.openai.com/*`
- `https://chatgpt.com/*`


插件会读取当前 ChatGPT 网页 DOM，用于：

- 识别会话结构
- 统计消息数量
- 判断内容是否离开窗口
- 测量内容高度
- 判断内容复杂度
- 生成本地诊断和调试记录

插件的诊断快照、调试记录功能仅用于本地碰到页面问题时辅助调试使用。
诊断快照和调试记录仅保存在本地；调试记录开始后会在面板和徽标上显示红色醒目的“调试进行中”警示状态，并且调试进行时不能复制或导出，必须停止后才能手动复制或导出。
调试记录默认使用 `redacted` 脱敏格式，不导出聊天正文、页面标题、原始 URL 路径、原始 turn/message ID、元素 `textContent`、`aria-label` 或 `title` 文本。
公开发布或提交问题反馈前，仍建议仔细检查诊断快照和调试记录，因为其中可能包含键名、输入长度、页面结构摘要、局部 DOM 标识、哈希关联 ID 等调试上下文。

## 调试记录

控制面板的“最近事件”页包含调试记录功能，用以记录页面结构的详细变化，精确查找问题所在。

需要注意：

- `Off` 模式下调试不可用。
- 如果要录制“无优化接管”的调试基线，应切到 `Monitor`。
- 调试进行时，面板和折叠徽标会以红色醒目状态提示正在调试，停止后恢复普通状态。
- 调试进行时不能复制或导出 JSON，停止调试后才可以复制或导出。
- 调试记录达到最大记录值后会自动停止，并保留最早的记录内容，不再滚动覆盖开头。

调试记录可以记录：

- 用户点击、输入、聚焦、选择等 DOM 事件元信息
- 按键名称、修饰键、输入类型和输入长度等有限输入元信息
- ChatGPT 页面结构变化摘要
- 插件同步、pipeline 阶段、降级/恢复和模式决策事件
- 插件样式写入的批次摘要
- 脱敏后的页面结构快照、差异和哈希关联 ID

当你遇到意外问题时，可以：

1. 打开控制面板。
2. 进入“最近事件”页。
3. 点击“开始调试”。
4. 复现问题。
5. 点击“停止调试”。
6. 手动导出 JSON 或复制 JSON。

注意：调试记录导出为 `schemaVersion: 2`、`privacyMode: "redacted"` 的 JSON。它默认不包含聊天正文或页面文本摘要，但仍可能包含结构化上下文、键名和输入长度。如果你需要发送给他人，请先检查文件内容。

## 项目结构

- `content/bootstrap/`：启动入口和命名空间初始化。
- `content/core/`：配置、国际化、日志、存储、诊断状态和通用工具。
- `content/dom/`：ChatGPT 页面识别、消息发现、布局测量、观察器和交互保护。
- `content/runtime/`：运行时调度、模式执行、性能指标、降级恢复和调试记录。
- `content/modes/`：优化模式注册中心和各模式实现。
- `content/ui/`：页面控制面板的布局、渲染、订阅和交互控制。
- `content/style/`：基础优化样式和模式样式。
- `icons/`：扩展图标。
- `_locales/`：浏览器扩展本地化文案。
